Поля чатов GetCourse

Модуль читает завершенные оплаченные заказы из customer-db.cdb_getcourse_orders, берет номер потока из актуальных чатов, берет VK/TG ссылки из Google Sheet 1zu1__XcKxJH8yC9ForDvibaUnKFCS1pxWHEjLgqlVXA и записывает поля обратно в заказ и в GetCourse addfields.

Логика

  • Источник заказов: записи с status=Завершен и payment_state=paid, обновленные не раньше настройки start_date.
  • Курс щенка определяется по фразе Первые шаги к воспитанию или слову Щенок; курс собаки — по Послушная собака или Современный собаковод. Если есть оба, выбирается щенок.
  • Тариф Стандарт записывается, но поля VK/TG ссылок остаются пустыми.
  • Поток записывается только цифрой из stream_number или названия чата, например 51.
  • Номер куратора берется из Google Sheets: ищется лист Щ{поток} для щенка или С{поток} для собаки, читается ячейка из настройки curator_cell. Сопоставление задается в настройке curator_map, например Ирина=Куратор 1;Слава=Куратор 2;Настасья=Куратор 3.
  • Если лист потока, credentials или понятное значение куратора не найдены, заказ получает статус pending_curator, ничего не записывается в customer-db/GetCourse, и следующий scan попробует снова.
  • VK/TG ссылки для не-стандартных тарифов берутся только из таблицы ссылок: щенок VK gid=65520414, щенок TG gid=1437498106, собака VK gid=443062527, собака TG gid=304757615. Если таблица недоступна или строки потока нет, заказ получает pending_chat_links и не записывается.
  • Панель обзора показывает все доступные потоки из таблицы ссылок в формате Поток / Куратор / Платформа / Название / Ссылка, без технических статусов модуля создания чатов.
  • Вкладка Потоки показывает локально кэшированный снимок листов потоков: список потоков, учеников по почтам, ссылку на пользователя GetCourse и ссылку на заказ. Сначала ссылки строятся по локальной customer-db. Если почты там нет, модуль может ограниченно добрать данные через GetCourse Export API и сохранить результат в локальный кэш.
  • Для обработки заказа текущий поток выбирается из course-chat-creator.runs: test_mode=0, status<>error, непустой link, сортировка по created_at DESC, id DESC.

Записываемые поля

  • Поток
  • Ссылка на чат ВК
  • Ссылка на чат ТГ
  • Номер куратора

Имена полей настраиваются в панели. В customer-db также пишутся служебные ключи chat_fields_* для диагностики.

GetCourse

Для записи в GetCourse нужны ENV GETCOURSE_ACCOUNT_NAME и GETCOURSE_API_TOKEN. Модуль пишет поля в пользователя через https://{account}.getcourse.ru/pl/api/users, action=add, system.refresh_if_exists=1 и user.addfields; в заказ — через https://{account}.getcourse.ru/pl/api/deals с deal.deal_number и deal.addfields.

Пользовательские поля отправляются по ID, потому что в аккаунте есть дубликат поля Поток: 10335965 = Поток, 12513209 = Ссылка на чат ВК, 12513210 = Ссылка на чат ТГ, 13834169 = Номер куратора.

Для вкладки Потоки GetCourse Export API используется только через фоновую очередь gc_export_lookup_jobs для email, которых нет в локальной customer-db. Открытие вкладки и refresh Google Sheets не вызывают GetCourse API. Worker берет по одному email, работает с коротким timeout, ведет журнал gc_export_api_calls и по умолчанию не делает больше 80 запросов за 2 часа, оставляя запас до лимита GetCourse 100. Результаты кэшируются в gc_export_lookup_cache.

Google Sheets

По умолчанию используется таблица 1NbKpXgHCJVE1dzpxeDLNGzSfz53vfK0joBRrNtIAaqk. Для приватной таблицы нужен service account JSON в GETCOURSE_CHAT_FIELDS_GOOGLE_CREDENTIALS_FILE, GOOGLE_SHEETS_SERVICE_ACCOUNT_FILE или путь в настройках панели.

Снимок учеников хранится в таблице модуля flow_students_cache. Настройка students_cache_minutes задает TTL, students_data_range — диапазон чтения строк учеников и верхних строк с куратором, по умолчанию A1:AC300. Если указан более узкий диапазон, модуль расширит его минимум до колонки AC, чтобы видеть объединенные заголовки кураторов. Ручная кнопка обновления принудительно перечитывает Google Sheets, а кнопка GC lookup отдельно ставит недостающие email в фоновую очередь.
