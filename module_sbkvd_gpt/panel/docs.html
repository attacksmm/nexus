SBKVD GPT

Модуль управления SBKVD GPT для команды Sobakovod. Внутри Nexus открывается экран доступов и настроек, а сам чат вынесен на отдельный полноэкранный адрес и генерирует ответы через модуль OpenRouter.

Адрес

/nexus/sbkvd-gpt/panel/      # настройки модуля в Nexus
/nexus/sbkvd-gpt/panel/chat/ # отдельный полноэкранный чат

Чат можно открывать отдельно от shell Nexus. Если пользователь не вошёл в GPT-аккаунт, чат покажет форму входа по логину.

Аккаунты и доступы

GPT-аккаунты отдельные от пользователей Nexus. Вход выполняется только по логину. Управление аккаунтами, доступными промптами и доступными моделями находится в основном экране модуля Nexus и доступно только пользователям Nexus с ролью admin.

Промпты и модели

Промпты берутся из Nexus file-storage: модуль показывает только .txt файлы, выданные конкретному GPT-аккаунту. Названия и группы промптов можно переименовывать в админке модуля без изменения самих файлов.

Модель, выбранная в интерфейсе чата, всегда передаётся в OpenRouter как явная модель и имеет приоритет над дефолтной моделью OpenRouter или моделью, назначенной prompt-файлу.

Контекст

При включённом «Хранить контекст» история сохраняется в базе sbkvd-gpt. В таблицы контекста модуля OpenRouter этот чат не пишет. При выключенном контексте ответ генерируется одним запросом и не сохраняется.

Зависимости

Для генерации должен быть активен модуль openrouter, а в нём должен быть настроен OPENROUTER_API_KEY.
