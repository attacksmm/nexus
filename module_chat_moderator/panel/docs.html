Модератор чатов

Модуль переносит простого модератора клубных чатов Sobakovod в Nexus и хранит историю действий в SQLite.

Runtime не стартует автоматически. Даже если модуль активен в Nexus, setup() только инициализирует БД и настройки.

Текущий инкремент

  • Добавлен безопасный skeleton модуля.
  • Добавлена БД истории действий и панель просмотра.
  • Перенесена простая TG/VK-модерация: детерминированные правила для мата, AI-анализ через OpenRouter, admin bypass, удаление негатива/скама и логирование.
  • VK-модерация не считает слово «блин», кинологические термины и обычные описания поведения собаки негативом без отдельного признака скама, уводящей ссылки или адресного оскорбления.
  • Приветствия и ответные сообщения выключены по умолчанию; модуль не обрабатывает возвраты и техподдержку как куратор учебных чатов.

ENV

Секреты можно задать двумя способами: через ENV или через вкладку «Настройки» в модуле. Если значение есть в ENV, оно всегда используется вместо значения из БД модуля.

NEXUS_CHAT_MODERATOR_RUNTIME_ENABLED=false
NEXUS_CHAT_MODERATOR_TG_ENABLED=false
NEXUS_CHAT_MODERATOR_VK_ENABLED=false
NEXUS_CHAT_MODERATOR_DRY_RUN=true
NEXUS_CHAT_MODERATOR_TG_SEND_RESPONSES=false
NEXUS_CHAT_MODERATOR_VK_SEND_RESPONSES=false
NEXUS_CHAT_MODERATOR_TG_WELCOME_ENABLED=false
NEXUS_CHAT_MODERATOR_VK_WELCOME_ENABLED=false
TELEGRAM_BOT_TOKEN_MODERATOR=
TELEGRAM_LOG_CHAT_ID=-1002852064172
VK_USER_TOKEN=
VK_LOG_CHAT_ID=
OPENROUTER_API_KEY=

Управление токенами

  • Источник env: значение задано в окружении Nexus и перекрывает модульную БД.
  • Источник module: значение сохранено в БД chat-moderator.db.
  • Источник default: используется безопасное значение по умолчанию, например модель OpenRouter.
  • Источник missing: токена нет, соответствующий runtime не стартует.

Пустое поле при сохранении не стирает текущий токен. Для удаления модульного значения используется кнопка «Очистить».

AI и prompt-файлы

AI-классификация идет через OpenRouter. Прямой ключ DeepSeek этому модулю не нужен.

VK-классификатор по умолчанию читает системный prompt из Nexus file-storage: prompts/moderator_prompt.txt, а модель берёт из настроек этого prompt-файла в модуле OpenRouter. До обращения к AI модуль применяет обязательные правила для мата и явных попыток увести людей в сторонние чаты. Если файл недоступен, модуль использует встроенный fallback prompt и не запускает runtime автоматически.

Правило работы

Не включать runtime в Nexus, пока старые модераторы работают в Sobakovod/Germany. Для проверки использовать отдельные тестовые чаты и запускать только один источник polling для конкретного TG токена.
