amoCRM -> Senler v1.0.0

Модуль принимает webhook amoCRM по сделкам, определяет настроенный статус, добавляет VK ID из utm_term сделки в группу Senler и записывает пользовательские переменные amo_responsible и amo_deal_id.

Webhook

URL берется во вкладке «Webhook». Если задан секрет, amoCRM должна отправлять запрос на URL с ?secret=.... Модуль принимает стандартный формат amoCRM application/x-www-form-urlencoded и JSON для ручного теста.

POST /amocrm-senler/api/webhook
Content-Type: application/x-www-form-urlencoded

leads[status][0][id]=123456
leads[status][0][pipeline_id]=111
leads[status][0][status_id]=142
leads[status][0][responsible_user_id]=987654

Связки

Связка определяет, в какую группу Senler добавлять подписчика. Для «Создана» используется событие создания сделки. Для «В работе», «Успех» и «Закрыто» в одной связке можно выбрать несколько статусов галочками из списка amoCRM: модуль сам сохраняет ID, но в панели показывает названия воронок и количество выбранных статусов.

Опция «Убирать из остальных» доступна для всех связок, кроме «Создана». Если она включена, после успешного добавления в группу текущей связки модуль удаляет подписчика из остальных активных групп amoCRM-связок через Senler subscribers/del. Удаление выполняется только из конкретных групп, без отписки от всех групп Senler.

Предустановленные группы:

  • 3713900 — AMO. Сделка создана
  • 3713899 — AMO. Сделка в работе
  • 3713898 — AMO. Закрыто и не реализованно
  • 3713878 — AMO. Успешно реализованные

Данные сделки

  1. VK ID берется из поля сделки utm_term. Имя поля можно изменить во вкладке «Настройки». Если поле содержит URL или строку с query-параметром utm_term=..., модуль извлечет значение из нее.
  2. Если webhook не содержит это поле, модуль запрашивает сделку через amoCRM API.
  3. Точное имя ответственного берется через amoCRM users API по responsible_user_id.
  4. После добавления в Senler вызывается vars/set для amo_responsible и amo_deal_id.
