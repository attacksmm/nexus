GetCourse -> amoCRM

Модуль принимает URL процесса GetCourse и создает или обновляет сделку amoCRM по схеме, близкой к Vakas-tools.

ENV

ПеременнаяНазначение
AMO_BASE_URLАккаунт amoCRM, например https://sobakovodpro.amocrm.ru.
AMO_ACCESS_TOKENТокен с доступом к сделкам, контактам, полям и воронкам.
GETCOURSE_AMO_WEBHOOK_SECRETНеобязательный внешний секрет. Если не задан, модуль сам создает секрет в SQLite.
GETCOURSE_AMO_CUSTOMER_DB_PATHНеобязательный путь к БД customer-db. Обычно не нужен: модуль сам находит runtime-БД старого getcourse-orders через customer-db.

Что записывается в сделку

Название: ЗАКАЗ №{number} | {object.user.name} | DD.MM.YYYY.

Воронка по умолчанию: 0. Автооплаты, статус 1. ПОПЫТКА АВТООПЛАТЫ, ответственный 6269974.

Поля сделки: № ГК, Пользователь в ГК, Ссылка на оплату, Заказ в ГК, Название тарифа, Оплачено, Осталось оплатить, Стоимость тарифа.

UTM пишутся в обе группы полей amoCRM: системные tracking-поля UTM_SOURCE/MEDIUM/CAMPAIGN/CONTENT/TERM и текстовые поля utm_source/utm_medium/utm_campaign/utm_content/utm_term. Также записываются yclid, _ym_uid, UTM_YM_UID, YM_CLIENT_ID, если такие поля есть в аккаунте.

Теги по умолчанию: GC и Автооплата. Контакт создается с именем, телефоном и email.

Связки

Во вкладке Связки отдельно настраиваются процессы created, partial и paid.

Для каждой связки задаются воронка, статус, ответственный за сделку, поведение при найденном дубле и задача amoCRM.

Кнопка Пауза связок останавливает только обработку связок. Сам модуль и панель остаются активными. При автосинхронизации из getcourse-orders новые строки не помечаются обработанными, пока пауза включена, поэтому после возобновления они уйдут в amoCRM.

Поведение при дубле: update обновляет найденную сделку, create всегда создает новую сделку, skip не трогает найденный дубль.

Поиск дубля настраивается в каждой связке отдельно. Можно искать среди сделок или среди контактов и выбрать несколько правил: поле amoCRM = значение из заказа GetCourse. Например, для сделок по умолчанию используются № ГК = number и ГК ID Заказа = order_id; для контактов можно выбрать PHONE = phone или EMAIL = email. Локальная карта уже обработанных заказов проверяется первой.

Задача создается через amoCRM POST /api/v4/tasks и привязывается к сделке. В тексте задачи доступны плейсхолдеры {number}, {order_id}, {name}, {phone}, {email}, {payment_link}, {title}, {cost_money}, {left_cost_money}, {payed_money}, {status}.

URL GetCourse

Если в GetCourse уже стоят процессы на /nexus/getcourse-orders/api/webhook/created|partial|paid, их можно не менять. Новый модуль автоматически читает таблицу getcourse_orders из customer-db, куда пишет старый модуль, и создает или обновляет сделку amoCRM.

При первом запуске существующие старые строки помечаются как уже увиденные без отправки в amoCRM, чтобы не создать исторические дубли. Новые и обновленные строки синхронизируются дальше автоматически.
