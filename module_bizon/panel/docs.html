Bizon365

Назначение

Модуль переносит логику sbkvd_bizon в Nexus: запускает headless runner Bizon365, держит комнаты открытыми, отслеживает консольные события модератора и пишет лог.

Runner запускается отдельным процессом и не использует основной Python env Nexus.

Скрипты автоответов

Вкладка Скрипты генерирует HTML для комнат Bizon365. Скрипт грузится с junior.sobakovod.pro/nexus, а ответы получает через публичный endpoint модуля Bizon, который server-side вызывает Nexus OpenRouter.

OpenRouter token в браузер не отдаётся. В комнате указываются prompt-файлы, например prompts/dog_gpt4.txt и prompts/dog_gpt4-2.txt; модель берётся из настроек prompt-файла в модуле OpenRouter.

Настройки

  • base_url, sec_key, список комнат и watchdog-настройки хранятся в SQLite модуля.
  • Логин и пароль Bizon365 можно задать через панель или через env BIZON365_LOGIN и BIZON365_PASS.
  • Пустое поле секретов при сохранении не стирает уже сохранённое значение.

Зависимости

Кнопка установки создаёт isolated venv в data-директории модуля, ставит playwright, Chromium и, если доступен sudo -n, системные библиотеки Chromium. Это не меняет зависимости Nexus.

API

  • GET /api/status - процесс, зависимости и публичная часть настроек.
  • GET /api/settings, POST /api/settings - чтение и сохранение настроек.
  • POST /api/deps/install - установка isolated runner dependencies.
  • POST /api/start, POST /api/stop, POST /api/restart - управление runner-процессом.
  • GET /api/logs, GET /api/deps/logs - хвост логов.
  • GET /api/scripts/rooms, PUT /api/scripts/rooms - конфиги скриптов по комнатам.
  • /api/public/process2, /api/public/user_lookup, /api/public/chat - публичные endpoints для JS в Bizon365.
