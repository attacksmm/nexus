amoCRM → DB v1.0.0

Модуль принимает webhook amoCRM по сделкам, догружает актуальные данные сделки через amoCRM API и сохраняет snapshot в customer-db таблицу amo_deals с названием «Сделки amoCRM».

ENV

ПеременнаяОписание
AMO_BASE_URLURL аккаунта amoCRM, например https://sobakovodpro.amocrm.ru.
AMO_ACCESS_TOKENТокен с доступом к сделкам, контактам и воронкам.
AMO_DB_WEBHOOK_SECRETНеобязательный секрет webhook. Если задан, имеет приоритет над настройкой в панели.
AMO_DB_CUSTOMER_DB_PATHНеобязательный путь к SQLite базе customer-db.

Webhook

URL берется во вкладке «Webhook». Если задан секрет, amoCRM должна отправлять запрос на URL с ?secret=.... Модуль принимает стандартный application/x-www-form-urlencoded формат amoCRM и JSON для ручного теста.

POST /nexus/amocrm-db/api/webhook?secret=...
leads[status][0][id]=123456
leads[status][0][pipeline_id]=111
leads[status][0][status_id]=142

Ответ для amoCRM всегда HTTP 200, чтобы amoCRM не отключала webhook при временной ошибке обработки. Ошибки фиксируются во вкладке «События».

Что сохраняется

Ключ записи в Базе клиентов: platform_id = deal_id. В custom_fields сохраняются сумма, воронка, статус, ответственный, даты, теги, контакты, телефоны, email, UTM, yclid, ym_uid, кастомные поля, raw snapshot сделки, ссылка на amoCRM и служебные поля синхронизации.
