OpenRouter API

Модуль принимает внешние запросы с Bearer token, читает промпты из file-storage и хранит локальный контекст по platform_id.

Генерация

POST /nexus/openrouter/api/chat
Authorization: Bearer <NEXUS_OPENROUTER_API_TOKEN>

{
  "platform_id": "vk_123",
  "conversation_id": null,
  "prompt": "prompts/avito_gpt1.txt",
  "message": "Вопрос клиента",
  "context": 2
}

context=0 не читает и не пишет историю, context=1 читает краткий контекст, но не пишет, context=2 читает краткий контекст и сохраняет вопрос/ответ, context=3 читает полный контекст и сохраняет, context=4 делает то же, что 3, и после ответа автоматически обновляет краткую сводку.

Если у клиента уже есть краткая сводка, при кратком чтении она используется в приоритете вместо истории сообщений.

Получение контекста

GET /nexus/openrouter/api/context/brief?conversation_id=or_conv_...
Authorization: Bearer <NEXUS_OPENROUTER_API_TOKEN>
GET /nexus/openrouter/api/context/full?conversation_id=or_conv_...
Authorization: Bearer <NEXUS_OPENROUTER_API_TOKEN>

Ручное пополнение

POST /nexus/openrouter/api/context/append
Authorization: Bearer <NEXUS_OPENROUTER_API_TOKEN>

{
  "platform_id": "vk_123",
  "conversation_id": "or_conv_...",
  "question": "1) вопрос пользователя",
  "answer": "1) ответ пользователя"
}

Настройки

OpenRouter API token можно указать во вкладке «Настройки». Непустое значение сохраняется в OPENROUTER_API_KEY и сразу загружается в окружение процесса Nexus.
