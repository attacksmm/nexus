OpenRouter API

Модуль принимает внешние запросы с собственным Bearer token, читает промпты из file-storage и хранит локальный контекст по platform_id.

Генерация

POST /nexus/openrouter/api/chat
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

{
  "platform_id": "vk_123",
  "conversation_id": null,
  "prompt": "prompts/avito_gpt1.txt",
  "message": "Вопрос клиента",
  "context": 2
}

context=0 не читает и не пишет историю, context=1 читает краткий контекст, но не пишет, context=2 читает краткий контекст и сохраняет вопрос/ответ, context=3 читает полный контекст и сохраняет, context=4 делает то же, что 3, и после ответа автоматически обновляет краткую сводку-рассказ о клиенте: кто клиент, что известно о собаке, какая проблема и что уже обсуждали.

Если у клиента уже есть краткая сводка, при кратком чтении она используется в приоритете вместо истории сообщений.

Avito → Salebot

Endpoint для Avito после генерации сохраняет контекст, разбивает ответ на части по 800 символов, записывает переменные в Salebot и вызывает callback.

POST /nexus/openrouter/api/avito
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

{
  "platform_id": "u2i-avito-user-id",
  "salebot_id": "967000000",
  "conversation_id": null,
  "prompt": "prompts/avito_gpt1.txt",
  "message": "Вопрос клиента",
  "context": 2
}

platform_id — ID пользователя в Avito. Также принимаются alias-поля avito_id, avito_user_id, user_id. salebot_id — числовой ID клиента Salebot, также можно отправлять client_id. Необязательные поля: split_size (по умолчанию 800) и callback_message (по умолчанию callback openai_answer). Для доставки нужен SALEBOT_API_KEY; если его нет, используется существующий SALEBOT_API_KEY_3.

conversation_id можно всегда отправлять из #{client.thread_id}. Endpoint принимает только ID вида or_conv_...; пустые значения, none, null, плейсхолдеры и старые OpenAI thread_... будут проигнорированы, после чего модуль сам возьмет активный чат по Avito platform_id или создаст новый.

В Salebot пишутся client.message, client.answer, JSON-копии, client.thread_id, client.openai_status, client.avito_id, client.salebot_id и client.answer1...client.answerN. Ссылки в частях ответа выделяются в отдельный answerN. Также создается/обновляется запись в customer-db таблице avito_clients.

Senler webhook

Для блока Senler «Обработать ответ в переменные» используйте отдельный endpoint. Он принимает те же поля, что /chat, но возвращает vars и glob_vars.

POST /nexus/openrouter/api/senler-chat
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

{
  "platform_id": 123456,
  "conversation_id": null,
  "prompt": "prompts/avito_gpt1.txt",
  "message": "Вопрос клиента",
  "context": 2
}
{
  "vars": [
    {"n": "ai_answer", "v": "Ответ модели"},
    {"n": "conversation_id", "v": "or_conv_..."},
    {"n": "platform_id", "v": "123456"}
  ],
  "glob_vars": []
}

Имена переменных можно переопределить полями answer_var, conversation_id_var, platform_id_var, model_var, summary_var, summary_error_var. Пустое имя отключает запись конкретной переменной.

Для Senler можно отправлять context=4: endpoint вернет ответ по краткой сводке о клиенте, сохранит вопрос/ответ и обновит сводку в фоне, не удерживая webhook ожиданием полной истории.

Получение контекста

GET /nexus/openrouter/api/context/brief?conversation_id=or_conv_...
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

Поля API

GET /nexus/openrouter/api/schema
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

Тестирование

Во вкладке «Протестировать» можно выбрать prompt, модель и пользователя/чат, контекст которого нужно только прочитать. Тестовый запрос не записывает вопрос и ответ в историю.

Все внешние /chat запросы логируются в модульный лог: старт, успешный ответ, HTTP-ошибки OpenRouter и ошибки валидации.

GET /nexus/openrouter/api/context/full?conversation_id=or_conv_...
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

Ручное пополнение

POST /nexus/openrouter/api/context/append
Authorization: Bearer <ТОКЕН МОДУЛЯ ИЗ НАСТРОЕК>

{
  "platform_id": "vk_123",
  "conversation_id": "or_conv_...",
  "question": "1) вопрос пользователя",
  "answer": "1) ответ пользователя",
  "update_summary": false
}

update_summary=true сразу пересобирает краткую сводку пользователя после добавления пары вопрос/ответ. По умолчанию false: пара только добавляется в историю conversation.

Настройки

Во вкладке «Настройки» отображается токен модуля для бота. OpenRouter API token можно указать там же: непустое значение сохраняется в OPENROUTER_API_KEY и сразу загружается в окружение процесса Nexus.
