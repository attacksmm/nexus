Заказы GetCourse

Модуль принимает webhook GetCourse, сохраняет заказ в таблицу customer-db getcourse_orders с названием «Заказы GetCourse» и по правилам добавляет VK ID в группы Senler.

Webhook

Основные endpoint для трёх отдельных процессов GetCourse: /getcourse-orders/api/webhook/created?secret=..., /getcourse-orders/api/webhook/partial?secret=..., /getcourse-orders/api/webhook/paid?secret=.... Каждый URL принудительно задаёт состояние заказа, независимо от сырого поля status. Старый /webhook оставлен для совместимости и определяет состояние по status.

order_id используется как platform_id. Если запись с таким platform_id уже есть, модуль обновляет её актуальными данными.

Оплата

Статусы GetCourse сопоставляются в настройках: оплаченные, частичные и неоплаченные. Senler запускается для paid, partial и unpaid, если есть подходящее активное правило.

Правила Senler

Каждое правило содержит группу Senler, режим условий AND или OR, набор условий и чекбокс «Убирать из остальных».

Если чекбокс включён, пользователь добавляется в целевую группу и удаляется только из групп других активных правил этого модуля. Если выключен, выполняется только добавление.

После успешного добавления модуль записывает переменные пользователя Senler: getcourse_deal_name, getcourse_deal_number, getcourse_deal_cost, getcourse_deal_payed, getcourse_deal_id, getcourse_deal_paylink, getcourse_user_email, getcourse_user_phone. Суммы пишутся только цифрами, без валюты; телефон нормализуется к виду +7XXXXXXXXXX; из названия заказа убирается хвостовой числовой ID GetCourse.

ENV

  • SENLER_ACCESS_TOKEN — токен Senler.
  • SENLER_GROUP_ID — ID группы ВКонтакте в Senler.
  • GETCOURSE_ORDERS_WEBHOOK_SECRET — необязательный секрет webhook, имеет приоритет над настройкой в UI.
  • GETCOURSE_CUSTOMER_DB_PATH — необязательный путь к SQLite БД customer-db, если автоопределение не подходит.
